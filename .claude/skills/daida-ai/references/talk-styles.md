# トークスクリプト文体定義

スピーカーノート（トークスクリプト）の文体をプレゼンの目的に合わせて選択する。

## casual（デフォルト）

**用途**: 社内LT、勉強会、コミュニティイベント

**特徴**:
- 「〜ですよね」「〜なんですけど」のような話し言葉
- 聴衆に語りかけるトーン
- 適度な間（…）を含む

**例**:
```
皆さん、こんにちは。今日はClaude Codeの話をしたいと思います。
...まず、従来の開発ってどうでしたっけ？コード書いて、テスト書いて、PR出して...
めっちゃ大変ですよね。でも、Claude Codeを使うと、これが劇的に変わるんです。
```

## keynote

**用途**: カンファレンス、大規模イベント

**特徴**:
- 丁寧だが堅すぎない「です・ます」調
- ストーリーテリング重視
- 数字やデータへの言及が多い
- 間（ポーズ）の指示を含む

**例**:
```
本日は「Claude Codeで変わる開発体験」というテーマでお話しさせていただきます。

（間）

まず、1つ質問です。皆さんの開発チームで、コードレビューにどれくらい時間をかけていますか？
調査によると、平均で開発時間の20%がレビューに費やされています。
```

## formal

**用途**: 企業プレゼン、提案、報告

**特徴**:
- 「〜でございます」を含む丁寧語
- データと根拠を重視
- 感情的な表現は控えめ
- 箇条書きの読み上げスタイル

**例**:
```
ご紹介いたします。本日のテーマは「AI支援による開発プロセスの最適化」でございます。
まず、現状の課題を3点お示しいたします。
1点目、開発速度のボトルネック。従来の手法では、1機能あたり平均3日を要しております。
```

## humorous

**用途**: 社内LT、カジュアルなイベント

**特徴**:
- ユーモアやツッコミを含む
- 自虐ネタOK
- テンポが速い
- 聴衆の笑い（想定）を含む

**例**:
```
はい、こんにちは！今日の発表は…（スライドめくる）…Claude Code！
え、知らない？大丈夫です、5分後には「もうこれなしでは生きていけない」って言ってますから。
冗談です。半分くらい。
```

## SKILL.md での使い方

SKILL.md内のStep 3（トークスクリプト生成）で、ユーザーに文体を選択させる。
Claude(LLM)はこのファイルの文体定義を参照してスクリプトを生成する。
